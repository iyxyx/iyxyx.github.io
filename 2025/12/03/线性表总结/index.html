数据结构

线性表总结

线性表

定义：线性表是n个具有相同数据类型数据元素的有限序列。

顺序表

  • 用连续的存储单元依次存储数据元素
  • 逻辑上相邻的元素物理上也相邻
  • 下标便于随机访问，时间复杂度为O(1)

链表

  • 用任意存储单元存储数据元素
  • 通过指针链接逻辑上相邻元素
  • 不支持随机访问，每次查找都需要遍历

栈

  • 后进先出的线性表
  • 只允许在一端（栈顶top）进行插入和删除操作
队列
  • 先进先出的线性表
  • 只允许在队尾rear插入，队首front删除

循环队列

  • 取模操作 每次移动队尾、队头指针都要通过取模运算
  • 判断队列满了(Q->rear + 1) % MAXSIZE == Q->front
  • 队头、队尾指针移动Q->rear = (Q->rear + 1) % MAXSIZE
总结

1.时间复杂度

操作 顺序表 链表 栈 队列
访问元素 O(1) O(n) O(1) O(1)
插入/删除 O(n) O(1) O(1) O(1)
查找元素 O(n) O(n) O(n) O(n)

2.空间复杂度

  • 顺序表：空间固定，可能浪费或不足
  • 链表：动态分配，更灵活但有指针开销
  • 栈/队列：根据实现方式决定（顺序/链式）

3.选择建议

  • 需要快速随机访问 → 顺序表
  • 需要频繁插入删除 → 链表
  • 需要后进先出特性 → 栈
  • 需要先进先出特性 → 队列
  • 不确定数据量 → 链表
  • 数据量固定 → 顺序表

4.应用场景

  1. 顺序表适用场景：
    · 数据量固定或变化不大
    · 需要频繁随机访问元素
    · 内存空间要求连续
  2. 链表适用场景：
    · 数据量变化频繁
    · 频繁在任意位置插入/删除
    · 内存空间不连续
  3. 栈适用场景：
    · 函数调用栈
    · 表达式求值
    · 括号匹配
    · 浏览器的前进后退
  4. 队列适用场景：
    · 任务调度
    · 消息队列
    · 广度优先搜索
    · 缓冲区管理
