树总结

树

定义：树是n(n>=0)个结点构成的有限集合。当n=0时，称为空树。 每个结点有且仅有一个父结点即一颗N个结点的树有N-1条边。 核心术语： - 父节点/子节点：直接相邻的层级节点； - 兄弟节点：同一父节点的子节点互称兄弟； - 叶子节点：无任何子节点的节点（树的最底层节点）； - 节点度：节点拥有的子节点数量（叶子节点度为0）； - 树的度：树中所有节点的最大度数； - 节点深度：从根节点到当前节点的层级数（根节点深度=1）； - 节点高度：自底向上计数（叶子节点高度=1）； - 树的深度/高度：树中所有节点的最大深度（与最大高度相等）； - 路径：从根节点到目标节点的节点序列，路径长度=节点数-1。 ## 二叉树 定义：二叉树是每个节点最多有2个子节点（左子树、右子树，次序不可颠倒）的树形结构，空树是特殊二叉树。

性质： 1.一个二叉树第i层的最大结点数为2i − 1,i>=1; 2.深度为k的二叉树有最大结点总数为2k − 1,k>=1; 3.若n0表示叶结点个数，n2是度为2的非叶子结点个数，那么满足关系n0=n2+1; 4.完全二叉树中，若节点总数为n，父节点i的左孩子为2i、右孩子为2i+1（i从1开始计数）。 - 满二叉树（完美二叉树）：所有非叶子节点均有2个子节点，叶子全在同一层（节点总数=2ⁿ-1，n为层数）。 - 完全二叉树：叶节点只能出现在最下层和次下层，且最下层的叶节点集中在树的左部（适合数组存储）。 - 二叉搜索树（BST）：左子树所有节点值＜根节点值，右子树所有节点值＞根节点值，中序遍历为升序。 - 平衡二叉树（AVL）：左右子树高度差（平衡因子）绝对值≤1，解决BST退化为链表的问题。

遍历方式： 1. 前序遍历：根 → 左 → 右（用途：复制二叉树、获取树结构）。

2. 中序遍历：左 → 根 → 右（用途：BST中获取升序序列）。 3. 后序遍历：左 → 右 → 根（用途：删除二叉树、计算节点路径和）。 4. 层序遍历（非递归）：按层级从左到右遍历（用途：求树高、判断完全二叉树）。

存储方式： - 顺序存储：用数组存储，适合完全二叉树（节省空间，访问高效）。 - 链式存储：用节点（含数据、左指针、右指针）串联，适合任意二叉树（灵活，增删方便）。 ### 线索二叉树 加上前驱和后继，使得像遍历双向链表一样去遍历二叉树。 1.头节点(自己创建)的lchild指向二叉树的根； 2.头节点的rchild指向遍历的最后一个节点； 3.第一个节点（叶子节点）的lchild指向头节点； 4.最后一个节点的rchild指向头节点。

#### 二叉搜索树（二叉排序树）（二叉查找树） 1.非空左子树的所有键值小于其根节点的键值； 2.非空右子树的所有键值大于其根节点的键值； 3.左、右子树都是二叉搜索树。 注： - 对二叉搜索树进行中序遍历，可得到一个从小到大的输出序列 - 从左分支逐层推下来查找到的的最小元素，从右分支逐层推下来查找到的的最大元素 ##### 平衡二叉树（AVL树） 定义： （1）任一结点的左、右子树均为AVL树 （2）根结点左、右子树高度差的绝对值不超过1 平衡调整（4种） 1. 左旋转（LL失衡） 适用场景： 失衡节点平衡因子=2，且其左子节点平衡因子≥0（左子树过深，左子树的左子树更重）。 2. 右旋转（RR失衡） 适用场景： 失衡节点平衡因子=-2，且其右子节点平衡因子≤0（右子树过深，右子树的右子树更重）。 3. 左右旋转（LR失衡） 适用场景： 失衡节点平衡因子=2，且其左子节点平衡因子=-1（左子树过深，左子树的右子树更重）。 4. 右左旋转（RL失衡） 适用场景： 失衡节点平衡因子=-2，且其右子节点平衡因子=1（右子树过深，右子树的左子树更重）。 ###### 堆和哈夫曼树 堆定义： 堆是特殊的队列，从堆中取出元素是依照元素的优先级大小，而不是元素进入队列的先后顺序。 堆是完全二叉树的逻辑结构，物理上用数组存储，满足：任意节点的值 ≥ 其所有子节点的值（大根堆）或 ≤ 其所有子节点的值（小根堆）。 - 优先级约束： ​- 大根堆：arr[i] ≥ arr[2i] 且 arr[i] ≥ arr[2i+1]（父 ≥ 左、右子）； ​- 小根堆：arr[i] ≤ arr[2i] 且 arr[i] ≤ arr[2i+1]（父 ≤ 左、右子）； - 结点关系： 设父节点下标为 i，则： 左子节点下标：2i； ​右子节点下标：2i+1； ​父节点下标：[i/2]（向下取整）。

哈夫曼树定义（不唯一）：“贪心思想+二叉树” 给定 n 个带权值的叶子节点，构造的所有叶子节点带权路径长度之和最小的二叉树，称为哈夫曼树（最优二叉树）。 - 权值：节点承载的数值（如字符出现频率、数据权重）； ​ - 路径长度：从根节点到目标节点的边数（非节点数）； ​ - 带权路径长度（WPL）：单个叶子节点的「权值×路径长度」之和，即 （wi 为第i个叶子权值，li为对应路径长度）；

  • 哈夫曼树特性：无度为1的节点，n个叶子节点对应 n-1个非叶子节点，非叶子节点权值=其左右子节点权值之和。

构造： 核心原则：权值越大的叶子节点，距离根节点越近（最小化总WPL）（循环选两个最小）。

哈夫曼编码（不唯一）：左子树记为 0 ，右子树记为 1 ，叶子节点的路径序列即为对应编码。

两两交换链表中的节点

两两交换链表中的节点

题目：给你一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。你必须在不修改节点内部的值的情况下完成。 示例1： 输入：head=[1,2,3,4] 输出：[2,1,4,3] ## 解题思路 1.情况分析：空链表、只有一个节点、节点数为奇数（最后一个节点不交换） 2.思路：整个问题可以拆成“两两交换”即“交换前两个节点”+“交换剩下的链表”，可用递归方法 ### 核心思想 - 对任意链表，先交换前两个节点，剩下的链表直接用递归函数处理 - 递归返回“子链表交换后的新头节点”，再拼接 #### 推导过程 1.对1->2->3->4,先处理1和2的交换，得到2->1 2.剩下的3->4用同样的方法交换（即递归）的得到4->3 3.将两个结果拼接得到最终结果2->1->4->3 ##### 代码实现（C语言）

线性表总结

线性表

定义：线性表是n个具有相同数据类型数据元素的有限序列。 ## 顺序表 - 用连续的存储单元依次存储数据元素 - 逻辑上相邻的元素物理上也相邻 - 下标便于随机访问，时间复杂度为O(1)

### 链表 - 用任意存储单元存储数据元素 - 通过指针链接逻辑上相邻元素 - 不支持随机访问，每次查找都需要遍历 #### 栈 - 后进先出的线性表 - 只允许在一端（栈顶top）进行插入和删除操作 ##### 队列 - 先进先出的线性表 - 只允许在队尾rear插入，队首front删除 循环队列 - 取模操作 每次移动队尾、队头指针都要通过取模运算 - 判断队列满了(Q->rear + 1) % MAXSIZE == Q->front - 队头、队尾指针移动Q->rear = (Q->rear + 1) % MAXSIZE ###### 总结 1.时间复杂度 | 操作 | 顺序表 | 链表 | 栈 | 队列 | | :——— | :—– | :—– | :—– | :—– | | 访问元素 | O(1) | O(n) | O(1) | O(1) | | 插入/删除 | O(n) | O(1) | O(1) | O(1) | | 查找元素 | O(n) | O(n) | O(n) | O(n) |

2.空间复杂度 - 顺序表：空间固定，可能浪费或不足 - 链表：动态分配，更灵活但有指针开销 - 栈/队列：根据实现方式决定（顺序/链式）

3.选择建议 - 需要快速随机访问 → 顺序表 - 需要频繁插入删除 → 链表 - 需要后进先出特性 → 栈 - 需要先进先出特性 → 队列 - 不确定数据量 → 链表 - 数据量固定 → 顺序表

4.应用场景 1. 顺序表适用场景： · 数据量固定或变化不大 · 需要频繁随机访问元素 · 内存空间要求连续 2. 链表适用场景： · 数据量变化频繁 · 频繁在任意位置插入/删除 · 内存空间不连续 3. 栈适用场景： · 函数调用栈 · 表达式求值 · 括号匹配 · 浏览器的前进后退 4. 队列适用场景： · 任务调度 · 消息队列 · 广度优先搜索 · 缓冲区管理

一、线性表

定义：线性表是具有相同数据类型的n个数据元素的有限序列。 序列：元素之间有序，中间元素都有且仅有一个直接前驱和一个直接后继。 当一个线性表中没有元素(长度为0)时，称为空表；表的起始位置称为表头，表的结束位置称为表尾。 ## 二、顺序表 用连续的存储空间(如数组)存储元素，支持随机访问(通过下标快速定位并访问随机元素)。 ### 三、结构定义以及初始化

注意：initList函数要传入一个SeqList类型的指针 实现对原结构体的修改，避免拷贝，节省内存(后续增删改查操作一致) #### 四、顺序表的核心操作(C语言实现) 增、删、改、查 1.尾部添加元素 - 先赋值再length++ 注意：把L->length当作数组下标 2.插入元素(指定位置) - 后移元素 - 最后记得length++ 3.删除元素(指定位置) - 前移元素 - 最后记得length-- 4.查找(给定一个参数e，查找第一次出现的位置) 5.修改 直接赋值 6.遍历 一个for循环遍历
