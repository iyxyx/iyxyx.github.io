两两交换链表中的节点

两两交换链表中的节点

题目：给你一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。你必须在不修改节点内部的值的情况下完成。
示例1：
输入：head=[1,2,3,4]
输出：[2,1,4,3]

解题思路

1.情况分析：空链表、只有一个节点、节点数为奇数（最后一个节点不交换）
2.思路：整个问题可以拆成“两两交换”即“交换前两个节点”+“交换剩下的链表”，可用递归方法

核心思想

  • 对任意链表，先交换前两个节点，剩下的链表直接用递归函数处理
  • 递归返回“子链表交换后的新头节点”，再拼接

推导过程

1.对1->2->3->4,先处理1和2的交换，得到2->1
2.剩下的3->4用同样的方法交换（即递归）的得到4->3
3.将两个结果拼接得到最终结果2->1->4->3

数据结构

线性表总结

线性表

定义：线性表是n个具有相同数据类型数据元素的有限序列。

顺序表

  • 用连续的存储单元依次存储数据元素
  • 逻辑上相邻的元素物理上也相邻
  • 下标便于随机访问，时间复杂度为O(1)

链表

  • 用任意存储单元存储数据元素
  • 通过指针链接逻辑上相邻元素
  • 不支持随机访问，每次查找都需要遍历

栈

  • 后进先出的线性表
  • 只允许在一端（栈顶top）进行插入和删除操作
队列
  • 先进先出的线性表
  • 只允许在队尾rear插入，队首front删除 循环队列
  • 取模操作 每次移动队尾、队头指针都要通过取模运算
  • 判断队列满了(Q->rear + 1) % MAXSIZE == Q->front
  • 队头、队尾指针移动Q->rear = (Q->rear + 1) % MAXSIZE
总结

1.时间复杂度

操作 顺序表 链表 栈 队列
访问元素 O(1) O(n) O(1) O(1)
插入/删除 O(n) O(1) O(1) O(1)
查找元素 O(n) O(n) O(n) O(n)

2.空间复杂度

  • 顺序表：空间固定，可能浪费或不足
  • 链表：动态分配，更灵活但有指针开销
  • 栈/队列：根据实现方式决定（顺序/链式）

3.选择建议

  • 需要快速随机访问 → 顺序表
  • 需要频繁插入删除 → 链表
  • 需要后进先出特性 → 栈
  • 需要先进先出特性 → 队列
  • 不确定数据量 → 链表
  • 数据量固定 → 顺序表

4.应用场景

  1. 顺序表适用场景：
    · 数据量固定或变化不大
    · 需要频繁随机访问元素
    · 内存空间要求连续
  2. 链表适用场景：
    · 数据量变化频繁
    · 频繁在任意位置插入/删除
    · 内存空间不连续
  3. 栈适用场景：
    · 函数调用栈
    · 表达式求值
    · 括号匹配
    · 浏览器的前进后退
  4. 队列适用场景：
    · 任务调度
    · 消息队列
    · 广度优先搜索
    · 缓冲区管理

一、线性表

定义：线性表是具有相同数据类型的n个数据元素的有限序列。
序列：元素之间有序，中间元素都有且仅有一个直接前驱和一个直接后继。
当一个线性表中没有元素(长度为0)时，称为空表；表的起始位置称为表头，表的结束位置称为表尾。

二、顺序表

用连续的存储空间(如数组)存储元素，支持随机访问(通过下标快速定位并访问随机元素)。
